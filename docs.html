Documentation

Everything you need to integrate and master NanoGrid in your application.

1. Installation

Because NanoGrid is entirely Vanilla JavaScript with zero dependencies, you don't need NPM or Webpack to use it.

Simply copy the nanogrid.js file into your project directory and include it in your HTML right before your closing </body> tag.

<!-- Include NanoGrid Core -->
<script src="nanogrid.js"></script>
Note: NanoGrid automatically injects its own minimal CSS styles into the DOM upon initialization. You do not need to link any external stylesheets.

2. Basic Usage

NanoGrid requires only a barebones HTML <table> structure. The library will automatically wrap it, apply styles, and build the controls (Search, Dropdowns, Pagination).

The HTML Structure

Define your columns inside the <thead>. Use the data-field attribute to tell NanoGrid which property from your JSON object maps to which column. You can also disable sorting or searching on specific columns using data-sortable="false" or data-searchable="false".

<table id="myTable">
  <thead>
    <tr>
      <th data-field="id">ID</th>
      <th data-field="name">Full Name</th>
      <th data-field="email" data-sortable="false">Email Address</th>
    </tr>
  </thead>
  <tbody></tbody>
</table>

The JavaScript Initialization

Instantiate the NanoGrid class, passing your table's ID and a configuration object.

document.addEventListener('DOMContentLoaded', () => {
  const users = [
    { id: 1, name: "Alice", email: "alice@example.com" },
    { id: 2, name: "Bob", email: "bob@example.com" }
  ];

  const grid = new NanoGrid('#myTable', {
    data: users,
    pageSize: 10
  });
});

3. Configuration API

You can customize the grid's behavior by passing properties into the config object.

Property Type Default Description
data Array [] Your array of JSON objects (used in Client-Side mode).
pageSize Number 10 The default number of rows displayed per page.
pageSizeOptions Array [5, 10, 20, 50] The available options in the "Rows per page" dropdown.
fontSize String '0.875rem' Globally controls the font size of the entire grid (e.g., '1rem', '14px').
searchable Boolean | Array true Toggles the global search bar OR restricts searching to specific column indices (e.g., [0, 2]).
checkboxes Boolean false Auto-injects a checkbox column for bulk row selection.
rowIdField String 'id' The JSON key used to uniquely identify rows when checkboxes are enabled.
exportable Boolean true Toggles the visibility of the unified Export dropdown menu.
serverSide Boolean false Enables backend processing (requires the ajax property).
processingText String 'Loading...' The text displayed alongside the spinner during API fetching.

4. Custom Columns (Badges & Buttons)

To render HTML (like Action Buttons or Status Badges) instead of raw text, use the columnDefs property mapped to your HTML's data-field.

new NanoGrid('#myTable', {
  data: users,
  columnDefs: {
    // Override the 'status' column
    status: (row) => {
      const color = row.status === 'Active' ? 'green' : 'gray';
      return `<span style="color: ${color}; font-weight: bold;">${row.status}</span>`;
    },
    
    // Create an entirely custom 'actions' column
    actions: (row) => `
      <button onclick="edit(${row.id})">Edit</button>
      <button onclick="del(${row.id})" style="color: red;">Delete</button>
    `
  }
});

5. Row Selection (Checkboxes)

NanoGrid can automatically inject a column of checkboxes for bulk actions without any changes to your HTML structure. It persistently tracks selections across paginations.

const grid = new NanoGrid('#myTable', {
  data: users,
  checkboxes: true,     // Enables the checkbox column
  rowIdField: 'uuid'    // Tell NanoGrid which property is the unique ID
});

// To retrieve the selected IDs later:
document.querySelector('#deleteBulk').addEventListener('click', () => {
  const selectedRows = grid.getSelected();
  console.log(selectedRows); // e.g., ["uuid-1", "uuid-5"]
});

6. Server-Side Processing (AJAX)

For large databases, enable serverSide: true and provide an async ajax function. NanoGrid will pass its state (page, sorting, search) to your function and automatically show a loading spinner.

new NanoGrid('#myTable', {
  serverSide: true,
  pageSize: 10,
  ajax: async (params) => {
    /* NanoGrid passes current state into 'params':
       { page: 1, pageSize: 10, search: "john", sortCol: "id", sortDir: "asc" }
    */
    
    const response = await fetch(`https://api.example.com/users?page=${params.page}&limit=${params.pageSize}&q=${params.search}`);
    const result = await response.json();
    
    // You MUST return data in this format:
    return {
      data: result.items,        // The array of rows
      total: result.totalCount   // The total rows in DB
    };
  }
});

7. Exporting Data

NanoGrid includes a native export engine that requires zero external libraries.

  • CSV: Parses the current dataset and triggers a file download.
  • Excel / Word: Wraps the HTML table data in an MS Office XML schema.
  • PDF: Utilizes browser-native print stylesheets to isolate the table, allowing users to save perfectly formatted PDFs directly from the Print dialog.
Pro Tip: Columns containing custom HTML (like Action buttons) or Checkboxes are automatically excluded from the exported files to keep your data clean.

8. Grid API Methods

Once you initialize your grid (e.g., const grid = new NanoGrid(...)), you can control it programmatically using the following methods.

reload(newData = null, resetPage = true)

Refreshes the grid data dynamically without reloading the page. Perfect for after you add, edit, or delete a record.

// SERVER-SIDE: Triggers a fresh AJAX call
grid.reload();

// SERVER-SIDE: Triggers an AJAX call but stays on current page
grid.reload(null, false);

// CLIENT-SIDE: Replaces the table with a new array
let newArray = [...oldArray, { id: 3, name: "Charlie" }];
grid.reload(newArray);

getSelected()

Returns an Array of the unique IDs of all currently checked rows.

const idsToDelete = grid.getSelected();

showProcessing(text) & hideProcessing()

Manually triggers NanoGrid's built-in loading spinner overlay. Useful if you are performing a heavy manual frontend task and want to block table interaction temporarily.

grid.showProcessing('Validating records...');

setTimeout(() => {
  grid.hideProcessing(); // Restores the table view
}, 2000);
