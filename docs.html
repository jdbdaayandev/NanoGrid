Documentation

A comprehensive, step-by-step guide to mastering NanoGrid—the zero-dependency, enterprise-ready data table.

Download nanogrid.js

1. Introduction

NanoGrid was engineered to solve a specific problem: bridging the gap between basic HTML tables and massive, bloated grid libraries that require Node.js, Webpack, or jQuery to function. It provides enterprise features in a single, lightweight file.

🚀
Zero Dependencies

No jQuery, no React, no build steps. It is a single, heavily optimized Vanilla JS file. Plug it in and it works.

🖥️
HTML-First Design

Define table behaviors directly via HTML attributes (e.g., data-sticky="left"). Less JavaScript, cleaner code.

⚡
Smart AJAX Loader

Native fetch() integration that intelligently overlays a loading spinner over your data while keeping headers readable.

🤝
Framework Agnostic

Automatically resolves CSS conflicts. Works flawlessly out-of-the-box inside AdminLTE, Bootstrap, or Tailwind projects.

Installation

Because NanoGrid is dependency-free, you do not need a package manager. Simply download the nanogrid.js file using the button above and include it in your HTML.

<!-- Place this before your closing </body> tag -->
<script src="/js/nanogrid.js"></script>
Note: NanoGrid automatically injects its own CSS theme into the DOM during initialization. You do not need to link any separate stylesheet files.

Quick Start

To initialize NanoGrid, you must provide a standard HTML <table>. NanoGrid parses the data-field attribute on your <th> tags to understand how to map your JSON data to specific columns.

1. Prepare your HTML

<table id="myTable">
  <thead>
    <tr>
      <th data-field="id"># ID</th>
      <th data-field="name">Full Name</th>
      <th data-field="email" data-sortable="false">Email Address</th>
    </tr>
  </thead>
  <tbody></tbody>
</table>

2. Initialize the Grid

document.addEventListener('DOMContentLoaded', () => {
  const dataset = [
    { id: 1, name: "Alice", email: "alice@example.com" },
    { id: 2, name: "Bob", email: "bob@example.com" }
  ];

  const grid = new NanoGrid('#myTable', {
    data: dataset
  });
});

Configuration API

The NanoGrid constructor accepts an object to globally modify the grid's behavior. Below are the available properties you can use to tailor the table to your application.

responsiveBooleantrueAuto-hides right-most columns into a collapsible accordion on small screens.

fontSizeString'0.875rem'Globally controls the font size using standard CSS values.
searchableBool|ArraytrueToggles the global search bar, or limits it to specific column indices.
checkboxesBooleanfalseInjects a checkbox column on the far left for bulk row selection.
rowIdFieldString'id'The unique key identifier in your JSON data.
serverSideBooleanfalseSwitches the table into AJAX mode, pushing processing to your backend.
ajaxStr|Obj|FnnullThe endpoint or configuration object used to fetch data from your server.

CSS Framework Compatibility

If you are injecting NanoGrid into an existing admin dashboard (such as AdminLTE or Bootstrap), NanoGrid has intelligent fallback mechanisms to prevent layout breaks.

  • Dropdown Safety: Standard Bootstrap uses data-toggle="dropdown". NanoGrid intentionally uses a custom data-ng-toggle="dropdown" to prevent Bootstrap's JavaScript from hijacking the grid's menus, avoiding console errors.
  • Border Normalization: If you accidentally leave the table-bordered class on your table, NanoGrid detects this and strips conflicting borders so the table remains clean.

Data Mapping & SQL Aliases

In real-world applications, your database column names (or SQL aliases) rarely match what you want your users to read on the screen. NanoGrid handles data mapping implicitly through your HTML.

1. Frontend Data Mapping

NanoGrid maps data strictly based on the data-field attribute. The text inside the <th> does not matter. If your backend query uses an alias like SELECT full_name AS author_alias, supply that exact alias to the data-field.

<!-- Expects { "author_alias": "John Doe" }, but displays "Author Name" visually -->
<th data-field="author_alias">Author Name</th>

2. Backend Sorting with Aliases

When a user clicks to sort an aliased column, NanoGrid sends sortCol: "author_alias" to your backend. Running an ORDER BY author_alias directly on your database will usually fail. You must map it back to the real database column in your backend code.

// Example (PHP): Translating the frontend alias back to the actual DB column
$sortCol = $request->input('sortCol');

$columnMap = [
    'author_alias' => 'users.actual_first_name_column'
];

$actualDbColumn = $columnMap[$sortCol] ?? $sortCol;
$query->orderBy($actualDbColumn, 'asc');

Responsive Collapse

Standard data tables break mobile layouts by forcing ugly horizontal scrollbars. By keeping responsive: true enabled, NanoGrid measures the container width. When columns run out of space, they are automatically hidden and a native + button is injected.

Sticky Columns & Column Filters

You can enable advanced UI features dynamically just by writing standard HTML attributes on your headers. No complex JavaScript required.

<!-- 1. Sticky: Keeps the column locked to the left edge while scrolling horizontally -->
<th data-field="id" data-sticky="left">ID</th>

<!-- 2. Filterable: Auto-generates a text search input specific to this column -->
<th data-field="role" data-filter="true">Role</th>

Row Expansion (Accordions)

If you have large blocks of text, nested lists, or secondary details that do not fit into a standard column, provide a callback function that returns HTML, and NanoGrid will generate a clickable + toggle for each row.

const grid = new NanoGrid('#myTable', {
  data: dataset,
  rowExpansion: (row) => `
    <div style="padding: 1rem; background: #f8fafc; border-radius: 8px;">
      <h4 style="margin-top:0;">More about ${row.name}</h4>
      <p>Contact Email: ${row.email}</p>
    </div>
  `
});

Row Checkboxes (Selection)

NanoGrid can automatically inject a column of checkboxes for bulk actions without requiring you to manually write the HTML for it. The grid also intelligently tracks selected rows across different pages.

const grid = new NanoGrid('#myTable', {
  data: dataset,
  checkboxes: true,     
  rowIdField: 'id' // Tells the grid what the primary key is
});

// Example: Retrieve the selected IDs on a button click
document.querySelector('#deleteBulk').addEventListener('click', () => {
  const idsToDelete = grid.getSelected();
  console.log("Deleting rows:", idsToDelete); // Output: [4, 12, 19]
});

Custom Renderers (HTML, Badges, Buttons)

Sometimes you need to render actual HTML elements (like a Status Badge or an Edit Button) instead of raw text. You can achieve this by passing functions into the columnDefs property, matching your data-field names.

new NanoGrid('#myTable', {
  data: dataset,
  columnDefs: {
    status: (row) => {
      const color = row.status === 'Active' ? 'green' : 'gray';
      return `<span style="color: ${color}; font-weight: bold;">${row.status}</span>`;
    },
    actions: (row) => `<button onclick="editRecord(${row.id})">Edit</button>`
  }
});

Server-Side & Payloads

For large datasets, client-side arrays will crash the browser. Enabling serverSide: true shifts pagination, searching, and sorting to your backend server.

1. The Simple GET Request

new NanoGrid('#myTable', {
  serverSide: true,
  ajax: '/api/users' 
});

2. POST Requests & Custom Payloads

If you need to send extra data (like passing a specific department ID filter to the backend), provide a configuration object to the ajax property.

new NanoGrid('#myTable', {
  serverSide: true,
  ajax: {
    url: '/api/users',
    method: 'POST',
    
    // CUSTOM PAYLOAD: Sent in the JSON body alongside the pagination state
    data: {
      department_id: 5,
      is_archived: false
    }
  }
});

CSRF Tokens & Authentication

Modern backend frameworks strictly require CSRF tokens for POST requests. You can easily inject these tokens via the headers object.

const csrfToken = document.querySelector('meta[name="csrf-token"]').getAttribute('content');

new NanoGrid('#myTable', {
  serverSide: true,
  ajax: {
    url: '/api/users',
    method: 'POST',
    headers: {
      'X-CSRF-TOKEN': csrfToken,
      'Authorization': `Bearer ${localStorage.getItem('token')}`
    }
  }
});

Export Engine

NanoGrid includes a powerful, native export engine requiring zero external plugins. The "Export" dropdown automatically appears if exportable: true.

  • Export CSV: Instantly parses the dataset and triggers a lightweight .csv download.
  • Export Excel: Wraps the HTML data in an MS Office XML schema for .xls output.
  • Print / PDF: Strips away unnecessary UI elements and triggers the browser's native print dialog.

Integration: Laravel (PHP)

Below is a standardized Laravel Controller recipe that natively interprets NanoGrid's request and returns the exact JSON structure required.

Important Note: Notice how the method returns a plain array return ['data' => ..., 'total' => ...]; instead of using response()->json(). This prevents Laravel from wrapping your data inside an "original" property object, which can cause NanoGrid to read your data as undefined! 
<?php

namespace App\Http\Controllers;

use App\Models\User;
use Illuminate\Http\Request;

class UserController extends Controller
{
    public function fetch(Request $request)
    {
        // 1. Capture NanoGrid Parameters
        $page = (int) $request->input('page', 1);
        $pageSize = (int) $request->input('pageSize', 10);
        $search = $request->input('search');
        $sortCol = $request->input('sortCol');
        $sortDir = $request->input('sortDir', 'asc');

        $query = User::query();

        // 2. Global Search Logic
        if (!empty($search)) {
            $query->where(function($q) use ($search) {
                $q->where('name', 'LIKE', "%{$search}%")
                  ->orWhere('email', 'LIKE', "%{$search}%");
            });
        }

        // 3. Sorting Logic
        if (!empty($sortCol) && $sortCol !== 'null') {
            $query->orderBy($sortCol, $sortDir);
        }

        // 4. Pagination
        $paginator = $query->paginate($pageSize, ['*'], 'page', $page);

        // 5. Return a Flat Array (Laravel auto-converts this to pure JSON)
        return [
            'data' => $paginator->items(),
            'total' => $paginator->total()
        ];
    }
}

Integration: CodeIgniter 4 (PHP)

If you are utilizing CodeIgniter 4, intercept NanoGrid's request using the built-in Query Builder. Make sure to use setJSON() to send a pure response.

<?php

namespace App\Controllers;

use App\Models\UserModel;

class UserController extends BaseController
{
    public function fetch()
    {
        // Support both GET and POST automatically
        $requestData = $this->request->is('post') 
                       ? $this->request->getJSON(true) 
                       : $this->request->getGet();

        $model = new UserModel();
        
        $search   = $requestData['search'] ?? '';
        $sortCol  = $requestData['sortCol'] ?? null;
        $sortDir  = $requestData['sortDir'] ?? 'asc';
        $page     = $requestData['page'] ?? 1;
        $pageSize = $requestData['pageSize'] ?? 10;

        if (!empty($search)) {
            $model->groupStart()
                  ->like('name', $search)
                  ->orLike('email', $search)
                  ->groupEnd();
        }

        if (!empty($sortCol) && $sortCol !== 'null') {
            $model->orderBy($sortCol, $sortDir);
        }

        $data = $model->paginate($pageSize, 'default', $page);
        $total = $model->pager->getTotal();

        // Return pure JSON response
        return $this->response->setJSON([
            'data' => $data,
            'total' => $total
        ]);
    }
}

Grid API Methods

Store your grid instance in a variable (e.g., window.myGrid = new NanoGrid(...)) to interact with it programmatically later.

reload(newData = null, resetPage = true)

Forces the grid to re-render. Ideal for refreshing the table after successfully deleting or inserting a record.

// SERVER-SIDE: Triggers a fresh AJAX call to the backend
myGrid.reload();

// SERVER-SIDE: Refresh data, but DO NOT return to Page 1
myGrid.reload(null, false);

getSelected()

When checkboxes: true is enabled, this method returns an Array of the unique identifiers (based on rowIdField) of all currently checked rows.

const idsToDelete = myGrid.getSelected(); // e.g., [4, 12, 19]
