Documentation

Everything you need to integrate and master NanoGrid in your application.

1. Installation

Because NanoGrid is entirely Vanilla JavaScript with zero dependencies, you don't need NPM or Webpack to use it.

Simply copy the nanogrid.js file into your project directory and include it in your HTML right before your closing </body> tag.

<!-- Include NanoGrid Core -->
<script src="nanogrid.js"></script>
Note: NanoGrid automatically injects its own CSS styles into the DOM upon initialization. You do not need to link any external stylesheets.

2. Basic Usage

NanoGrid requires only a barebones HTML <table> structure. The library will automatically wrap it, apply styles, and build the controls (Search, Dropdowns, Pagination).

The HTML Structure

Define your columns inside the <thead>. Use the data-field attribute to tell NanoGrid which property from your JSON object maps to which column.

<table id="myTable">
  <thead>
    <tr>
      <th data-field="id">ID</th>
      <th data-field="name">Full Name</th>
      <th data-field="email" data-sortable="false">Email Address</th>
    </tr>
  </thead>
  <tbody></tbody>
</table>

The JavaScript Initialization

document.addEventListener('DOMContentLoaded', () => {
  const users = [
    { id: 1, name: "Alice", email: "alice@example.com" },
    { id: 2, name: "Bob", email: "bob@example.com" }
  ];

  const grid = new NanoGrid('#myTable', {
    data: users
  });
});

4. Responsive Collapse (Mobile Friendly)

By default, NanoGrid uses an intelligent responsive engine (responsive: true). When the screen width gets too small, NanoGrid automatically hides the right-most columns and injects a + button on the left to reveal hidden data as a vertical list.

5. Sticky Columns & Per-Column Filters

Keep important columns in view using data-sticky="left" or add column-specific search boxes using data-filter="true".

<!-- Sticky Column -->
<th data-field="id" data-sticky="left">ID</th>

<!-- Filterable Column -->
<th data-field="role" data-filter="true">Role</th>

6. Row Expansion (Accordion Details)

Pass a function to rowExpansion to return extra HTML details when a row is clicked.

const grid = new NanoGrid('#myTable', {
  data: users,
  rowExpansion: (row) => `
    <div style="padding: 1rem;">
      <h4>Additional Info for ${row.name}</h4>
      <p>Email: ${row.email}</p>
    </div>
  `
});

7. Row Selection (Checkboxes)

NanoGrid persistently tracks selections across paginations.

const grid = new NanoGrid('#myTable', {
  data: users,
  checkboxes: true,     
  rowIdField: 'uuid'    
});

// To retrieve IDs: grid.getSelected();

8. Custom Columns (Badges & Buttons)

new NanoGrid('#myTable', {
  data: users,
  columnDefs: {
    status: (row) => {
      const color = row.status === 'Active' ? 'green' : 'gray';
      return `<span style="color: ${color}; font-weight: bold;">${row.status}</span>`;
    },
    actions: (row) => `<button onclick="del(${row.id})">Delete</button>`
  }
});

9. Server-Side Processing (AJAX)

Enable serverSide: true and provide an async ajax function. NanoGrid passes its state (page, sorting, search, colFilters) to your backend.

new NanoGrid('#myTable', {
  serverSide: true,
  ajax: async (params) => {
    /* 'params' contains:
       { page: 1, pageSize: 10, search: "john", sortCol: "id", sortDir: "asc", colFilters: {} }
    */
    const query = new URLSearchParams(params).toString();
    const response = await fetch(`/api/users?${query}`);
    const result = await response.json();
    
    // You MUST return this format:
    return { data: result.data, total: result.total };
  }
});

10. Exporting Data

NanoGrid includes a native export engine for CSV, Excel, Word, and PDF. Custom columns (buttons) and checkboxes are automatically stripped out of exports to ensure clean data.

11. Integration: Laravel (PHP)

Using NanoGrid's serverSide: true with a Laravel backend is incredibly easy. NanoGrid's payload maps perfectly to Laravel's Eloquent queries and paginators.

Controller Example

Here is a copy-paste template for your Laravel Controller to handle global searches, sorting, and pagination.

<?php

namespace App\Http\Controllers;

use App\Models\User;
use Illuminate\Http\Request;

class UserController extends Controller
{
    public function getGridData(Request $request)
    {
        $query = User::query();

        // 1. Handle Global Search
        if ($search = $request->input('search')) {
            $query->where(function($q) use ($search) {
                $q->where('name', 'LIKE', "%{$search}%")
                  ->orWhere('email', 'LIKE', "%{$search}%");
            });
        }

        // 2. Handle Sorting
        if ($sortCol = $request->input('sortCol')) {
            $sortDir = $request->input('sortDir', 'asc');
            $query->orderBy($sortCol, $sortDir);
        }

        // 3. Handle Pagination
        $page = $request->input('page', 1);
        $pageSize = $request->input('pageSize', 10);
        
        $paginator = $query->paginate($pageSize, ['*'], 'page', $page);

        // 4. Return NanoGrid's required JSON structure
        return response()->json([
            'data' => $paginator->items(),
            'total' => $paginator->total()
        ]);
    }
}

12. Integration: CodeIgniter 4 (PHP)

If you are using CodeIgniter 4, you can handle NanoGrid's request using the built-in Query Builder and Pagination engine.

Controller Example

<?php

namespace App\Controllers;

use App\Models\UserModel;

class UserController extends BaseController
{
    public function getGridData()
    {
        $model = new UserModel();
        
        $search   = $this->request->getGet('search');
        $sortCol  = $this->request->getGet('sortCol');
        $sortDir  = $this->request->getGet('sortDir') ?? 'asc';
        $page     = $this->request->getGet('page') ?? 1;
        $pageSize = $this->request->getGet('pageSize') ?? 10;

        // 1. Handle Global Search
        if (!empty($search)) {
            $model->groupStart()
                  ->like('name', $search)
                  ->orLike('email', $search)
                  ->groupEnd();
        }

        // 2. Handle Sorting
        if (!empty($sortCol)) {
            $model->orderBy($sortCol, $sortDir);
        }

        // 3. Handle Pagination
        $data = $model->paginate($pageSize, 'default', $page);
        $total = $model->pager->getTotal();

        // 4. Return NanoGrid's required JSON structure
        return $this->response->setJSON([
            'data' => $data,
            'total' => $total
        ]);
    }
}

13. Grid API Methods

Control your grid programmatically after initialization.

reload(newData = null, resetPage = true)

// SERVER-SIDE: Triggers a fresh AJAX call
grid.reload();

// SERVER-SIDE: Triggers an AJAX call but stays on current page
grid.reload(null, false);

// CLIENT-SIDE: Replaces the table with a new array
grid.reload(newArray);

getSelected()

Returns an Array of the unique IDs of all currently checked rows.

const idsToDelete = grid.getSelected();

showProcessing(text) & hideProcessing()

Manually triggers NanoGrid's built-in loading spinner overlay.

grid.showProcessing('Validating records...');
setTimeout(() => grid.hideProcessing(), 2000);
